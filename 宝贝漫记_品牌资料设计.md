## 一、小程序命名设计

### 1.1 命名策略分析

基于以下心理学原则选择名字：

| 原则 | 说明 | 应用 |
|------|------|------|
| **处理流畅性** | 名字越容易读/记，好感度越高 | 2-4个字，无生僻字，朗朗上口 |
| **情感锚定** | 名字本身要触发目标情感 | "宝贝""漫"直接关联"孩子+漫画" |
| **社交可述性** | 别人提起这个产品时，名字好说出口 | "我用宝贝漫记给娃画漫画" ← 自然 |
| **搜索唯一性** | 在微信搜索中能唯一定位 | 避免通用词组合（如"宝宝相册"） |
| **品类暗示** | 名字暗示产品做什么 | "漫"=漫画，"记"=记录 |

### 1.2 推荐名称方案

#### 首选：宝贝漫记

| 维度 | 评分 | 理由 |
|------|------|------|
| 好记程度 | ⭐⭐⭐⭐⭐ | 4个字，读起来有节奏感（宝贝/漫记） |
| 情感温度 | ⭐⭐⭐⭐⭐ | "宝贝"=爱称，"记"=珍贵的记忆 |
| 功能暗示 | ⭐⭐⭐⭐⭐ | 宝贝+漫（漫画）+记（记录）= 给宝贝做漫画记录 |
| 传播性 | ⭐⭐⭐⭐ | "我用宝贝漫记" — 说出来自然不尬 |
| 搜索唯一性 | ⭐⭐⭐⭐⭐ | "宝贝漫记"组合具有高度唯一性 |
| 视觉想象 | ⭐⭐⭐⭐ | 听到名字就能想到"一本宝宝的漫画日记" |

**口碑传播模拟：**
```
妈妈A发朋友圈 → 评论区：
"这漫画太可爱了！怎么做的？"
"用宝贝漫记，说一句话就能生成！"  ← 4个字，记住了

线下场景：
"你用什么记录宝宝的？"
"宝贝漫记，一个小程序，特别好玩"  ← 好说出口
```

#### 备选方案

| 名称 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| 画个宝宝 | 极度口语化，门槛低 | 略casual，不够品牌感 | ⭐⭐⭐⭐ |
| 漫画宝贝 | 直接，功能明确 | 太直白，缺乏想象空间 | ⭐⭐⭐ |
| 小漫记 | 简短，萌感 | "小"字太多小程序用，不唯一 | ⭐⭐⭐ |
| 时光漫画 | 有意境 | 偏通用，不够聚焦母婴 | ⭐⭐⭐ |
| 宝贝画传 | 有文化感 | "传"字让人联想"传记"，太正式 | ⭐⭐ |

### 1.3 命名的病毒营销设计

**名字本身就是传播句的一部分：**

```
传播链路设计：

用户看到朋友分享的漫画
  ↓
"好可爱！这是什么？"
  ↓
"宝贝漫记，微信搜就有"  ← 名字即搜索词
  ↓
新用户搜索"宝贝漫记"
  ↓
唯一结果，直接进入

关键设计：
- 名字 = 搜索关键词（不需要记URL或扫码）
- 4个字 = 一句话就能传递
- "宝贝"唤起妈妈身份认同
- "漫记"暗示产出物（漫画日记）
```

## 二、账号头像/Logo设计

### 2.1 设计心理学原则

| 原则 | 应用 |
|------|------|
| **婴儿图式效应 (Baby Schema)** | 使用圆润、大眼、大头比例的元素 → 人类本能地觉得可爱 |
| **暖色优先** | 橘色/粉色/米色 → 让人感到温暖安全（冷色=距离感） |
| **简单辨识** | 在小程序列表(44x44px)中也能一眼认出 |
| **品类联想** | 看到logo就知道"这跟宝宝+画画有关" |
| **情感共鸣** | 妈妈看到后觉得"这是给我和宝宝的" |

### 2.2 Logo方案推荐

#### 方案A（首推）：翻开的绘本 + 宝宝侧脸

```
视觉描述：
┌─────────────────────────┐
│                         │
│    ┌───────────┐        │
│    │  📖       │        │
│    │ (翻开的    │        │
│    │  小绘本,   │        │
│    │  从绘本中   │        │
│    │  探出一个   │        │
│    │  圆脸宝宝   │        │
│    │  的剪影)   │        │
│    └───────────┘        │
│                         │
│  色调：暖橘#FF8A65       │
│  背景：奶白#FFFDF7       │
│                         │
└─────────────────────────┘

设计细节：
- 绘本用圆角表现，不要尖角
- 宝宝的头只露出上半部（大圆眼+蓬松头发轮廓）
- 像"宝宝从故事书里探出头来偷看你"
- 整体外形是一个圆形（适配头像框）
- 主色暖橘，搭配奶白底
```

#### 方案B：手绘气泡 + 星星

```
视觉描述：
- 一个对话气泡的形状（圆润，不是尖角的）
- 气泡里有一颗小星星和一个简笔小人（宝宝）
- 气泡本身像"宝宝说的话变成了画"
- 色调：暖橘主色 + 蜜糖黄点缀

含义：
- 气泡 = 宝宝说了一句话
- 星星 = 这句话变成了闪亮的漫画
- 整体表达"话语→画面"的魔法
```

#### 方案C：铅笔 + 爱心

```
视觉描述：
- 一支圆润的铅笔在画画
- 铅笔画出的线条形成一个爱心形状
- 爱心里有一个小圆脸（宝宝轮廓）
- 色调：暖橘笔身 + 樱粉爱心

含义：
- 铅笔 = AI在画画
- 爱心 = 充满爱
- 宝宝在爱心里 = 记录的核心是宝宝
```

## 三、小程序介绍/简介文案

### 3.1 文案心理学框架

```
好的产品介绍 = 
  ① 共情痛点（让妈妈觉得"说的就是我"）
  + ② 魔法承诺（极简描述产品做什么）
  + ③ 行动暗示（让人想立刻试试）
```

### 3.2 一句话简介（小程序搜索展示，限30字）

| 版本 | 文案 | 字数 | 心理机制 |
|------|------|------|---------|
| **首推** | 说一句话，AI帮你画出宝宝的搞笑温馨瞬间 | 19字 | 极简描述+情感场景 |
| 备选1 | 来不及拍？说出来，AI把宝宝的故事画成漫画 | 19字 | 痛点切入+解决方案 |
| 备选2 | 一句话变漫画，记录宝贝的每个高光时刻 | 16字 | 魔法感+价值感 |
| 备选3 | 宝宝说的话太可爱了，让AI画下来吧 | 15字 | 对话感，像朋友推荐 |

### 3.3 详情页介绍（小程序详情，限120字）

#### 首推版本

```
宝宝把碗扣头上说自己是宇航员？这么可爱的瞬间来不及拍照？

打开宝贝漫记，说一句话，AI秒变4格漫画。
18种风格随心选，温馨、搞笑、热血应有尽有。
还能生成宝宝专属角色卡，让每幅画里的宝宝都像TA。

现在就记录第一个瞬间吧 ✨
```

**心理机制拆解：**
- 第1段：具体场景唤起共鸣（"我家宝宝也这样！"）
- 第2段：核心功能一句话说清
- 第3段：差异化功能（风格多、角色卡）
- 第4段：行动召唤（CTA）

#### 情感版（更走心）

```
那些来不及按下快门的瞬间，
宝宝说过的搞笑的话、做过的温馨的事，
只要你还记得，AI就能帮你画下来。

一句话 → 一幅漫画 → 一段永远不会褪色的回忆。

宝贝漫记，让爱有迹可循 💝
```

## 六、品牌声音（Brand Voice）

### 6.1 品牌人格定义

```
「宝贝漫记」的人格是：

一位温暖的、有艺术气质的年轻妈妈朋友
- 她理解你带娃的手忙脚乱
- 她欣赏你记录生活的用心
- 她不会给你压力，但会温柔地鼓励你
- 她说话轻松幽默，偶尔卖萌
- 她的审美比你好一点，但不会让你自卑

NOT:
- ❌ 不是冰冷的科技产品（"AI智能生成系统"）
- ❌ 不是高高在上的品牌（"引领行业的..."）
- ❌ 不是过于幼稚的（"宝宝最棒棒！"）
- ❌ 不是焦虑贩卖的（"别的妈妈都在用..."）
```

### 6.2 文案语气规范

| 场景 | ✅ 要这样说 | ❌ 不要这样说 |
|------|-----------|-------------|
| 欢迎 | "欢迎回来，小土豆的妈妈🌻" | "欢迎使用本系统" |
| 引导 | "宝宝今天又做了什么趣事？" | "请输入内容" |
| 成功 | "画好啦！快看看～" | "生成成功" |
