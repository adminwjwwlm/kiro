## 一、产品概述

### 1.1 产品愿景

让每一个来不及按下快门的亲子瞬间，都能被AI还原为一幅温暖的漫画，并在时光流转中化为动人的成长影像。

### 1.2 一句话定义

**「宝贝漫记」** 是一款AI驱动的微信小程序，父母只需输入一段文字描述（宝宝说的话或当时的场景），即可在数秒内生成风格多样的剧情漫画，并支持一键分享与长期收藏。积累的漫画还可生成动态回忆视频，让静态画面"活"起来。

### 1.3 目标用户画像

| 维度 | 描述 |
|------|------|
| 年龄 | 25-40岁 |
| 身份 | 新手父母（核心用户80%为妈妈） |
| 设备 | iPhone / Android，重度微信用户 |
| 心理诉求 | 记录宝宝成长、炫娃、获得社交认同 |
| 使用场景 | 碎片化时间（通勤、午休、睡前） |
| 痛点 | 很多温馨/搞笑瞬间来不及拍照，文字记录缺乏感染力 |

### 1.4 产品定位矩阵

```
             高情感价值
                 │
    「宝贝漫记」 ●│
                 │            ○ 亲宝宝/宝宝树（记录类）
                 │
  ──────────────┼──────────────── 高创作门槛
                 │
     ○ 美图秀秀  │         ○ Midjourney/SD
    （简单修图）  │         （专业AI绘画）
                 │
             低情感价值
```

### 1.5 核心价值主张

1. **极简创作**：说一句话即可生成漫画，零学习成本
2. **情感连接**：不是冰冷的AI工具，而是温暖的家庭记忆载体
3. **社交货币**：高颜值漫画让分享变得自然且高级
4. **成长沉淀**：时间线积累形成不可替代的"漫画版成长日记"
5. **时光重现**：AI让静态漫画动起来，串联成长轨迹形成动人的回忆视频

## 五、角色库 / 资产库（V1.1核心功能）

### 5.1 功能概述

角色库是「宝贝漫记」的核心差异化功能。用户上传宝宝的真人照片后，AI会生成该宝宝在不同艺术风格下的"角色卡"。此后所有漫画生成都可以使用专属角色，保证漫画中的宝宝"像自己家的孩子"。

### 5.5 角色库使用场景

| 场景 | 说明 |
|------|------|
| 创作漫画时选角色 | 生成漫画时可选"用小土豆的形象"，AI自动将角色融入画面 |
| 多角色互动 | 支持同时选择"小土豆+妈妈+爸爸"出现在同一漫画中 |
| 风格统一 | 选定角色后，漫画风格自动匹配角色卡的风格 |
| 角色成长 | 每隔几个月提醒更新照片，角色卡随宝宝长大而变化 |
| 角色卡分享 | 角色卡本身就是可分享的精美内容 |

### 5.6 角色库技术要求

| 要求 | 规格 |
|------|------|
| 照片要求 | 支持jpg/png，512px以上，1-5张 |
| 面部检测 | 自动检测面部，裁剪对齐 |
| 生成时间 | 全套9个风格角色卡 ≤ 30秒 |
| 一致性 | 同一角色在不同漫画中保持面部特征一致 |
| 隐私安全 | 照片仅用于角色生成，不存储原片（生成后可删除） |
| 存储限制 | 免费用户最多3个角色，会员无限制 |

### 5.7 角色库数据模型

```

├── id: string
├── name: string (昵称)

├── source_photos: Photo[] (原始照片，可选保留)
```

### 7.6 回忆视频（V1.5试水 → V2.0完整版）

#### 7.6.1 功能概述

回忆视频是「宝贝漫记」从"记录工具"升维为"时光机器"的关键功能。通过AI将静态漫画逐帧动态化，配合智能转场和成长时间轴串联，自动生成一部有叙事感的「成长动画短片」。

**核心逻辑：不是幻灯片，是让漫画"活"起来**

```
漫画1 → [AI动态化：人物眨眼、挥手、场景微动] → 5秒动态片段
    ↓
[智能转场：翻页/画笔绘制/水彩晕染]
    ↓
漫画2 → [AI动态化] → 5秒动态片段
    ↓
[时间轴标注 + BGM + 字幕]
    ↓
= 完整的「成长动画短片」(30秒-3分钟)
```

### 9.2 AI生成核心链路

```
用户输入文字
    │
    ▼
┌─────────────────┐
│ 1. 文本理解      │  LLM (通义千问/GPT-4)
│    - 提取场景    │  输出：场景描述、人物、情绪、动作
│    - 拆解分镜    │  输出：4格分镜脚本(每格的prompt)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 2. 角色注入      │  如果用户有角色卡
│    - 加载embedding│  将角色特征注入到每个分镜prompt
│    - 保持一致性   │  使用IP-Adapter/InstantID技术
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 3. 图像生成      │  Stable Diffusion XL / Flux
│    - 按风格LoRA  │  加载对应风格的LoRA权重
│    - 批量生成4图 │  并行生成加速
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 4. 后处理合成    │  自动化排版
│    - 添加气泡    │  对话气泡+文字排版
│    - 4格拼接     │  统一尺寸、添加间距
│    - 加水印/标签 │  日期、年龄、品牌标识
└────────┬────────┘
         │
         ▼
    最终漫画输出
```

### 9.3 关键技术选型

| 模块 | 推荐方案 | 备选方案 | 选型理由 |
|------|---------|---------|---------|
| 小程序框架 | Taro 3.x (React) | uni-app | 跨端能力强，生态成熟 |

| 缓存 | Redis Cluster | - | 热点数据、限流计数 |
| 数据库 | MySQL 8.0 | PostgreSQL | 业务数据存储 |
| 向量数据库 | Milvus | Pinecone | 角色embedding存储与检索 |
| 消息队列 | RocketMQ | Kafka | AI任务+视频任务异步处理 |
| 监控 | Prometheus + Grafana | - | 系统&业务监控 |

## 十二、风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
