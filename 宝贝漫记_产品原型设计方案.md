## 一、设计理念与融合策略

### 1.1 为什么选择Don Norman为核心

「宝贝漫记」不是一个效率工具，而是一个**情感载体**。妈妈打开它的动机不是"完成任务"，而是"留住爱"。

Don Norman的情感化设计三层次模型完美映射本产品：

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   反思层 (Reflective) ← Don Norman核心                   │
│   "我在认真记录宝宝成长，我是个用心的妈妈"                 │
│   ↑ 意义感、成就感、社交认同                              │
│                                                         │
│   ─────────────────────────────────────────             │
│                                                         │
│   行为层 (Behavioral) ← 张小龙极简哲学                    │
│   "好顺手！说一句话就出来了，太方便了"                     │
│   ↑ 流畅、无摩擦、高效、符合直觉                          │
│                                                         │
│   ─────────────────────────────────────────             │
│                                                         │
│   本能层 (Visceral) ← Jony Ive质感美学                    │
│   "好好看！好温暖！好想点进去！"                           │
│   ↑ 色彩、质感、排版、第一印象                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 三位大师的融合分工

| 设计层面 | 主导思维 | 具体负责 |
|---------|---------|---------|
| 视觉语言 & 第一印象 | Jony Ive | 色彩体系、圆角规范、阴影质感、字体选择 |
| 操作流程 & 信息架构 | 张小龙 | 页面层级、操作步数、"用完即走"节奏 |
| 情感节点 & 微交互 | Don Norman | 等待体验、成就反馈、文案温度、分享仪式感 |

### 1.3 十条设计原则

| 序号 | 原则 | 来源 | 具体应用 |
|------|------|------|---------|
| 1 | 一屏一事 | 张小龙 | 每个页面只让用户做一个决策 |
| 2 | 情感优先于功能 | Don Norman | 所有文案用"妈妈视角"，不用技术语言 |
| 3 | 减法即美 | Jony Ive | 视觉元素能少则少，留白即呼吸 |
| 4 | 即时反馈 | Don Norman | 每个操作0.3秒内有视觉/触觉响应 |
| 5 | 不打断心流 | 张小龙 | 创作过程中零弹窗、零广告 |
| 6 | 质感传递信任 | Jony Ive | 圆润、柔和、温暖的视觉语言=安全感 |
| 7 | 让用户成为主角 | Don Norman | 产品退后，宝宝的故事在前 |
| 8 | 分享是自然延续 | 张小龙 | 分享不是推销，是"我想给家人看看" |
| 9 | 时间赋予意义 | Don Norman | 每张漫画都带时间戳，积累即价值 |
| 10 | 触感即引导 | Jony Ive | 按钮大小、位置暗示优先级 |

### 2.2 圆角与阴影规范

| 元素 | 圆角 | 阴影 | 设计意图 |
|------|------|------|---------|
| 卡片（漫画作品） | 16px | 0 4px 12px rgba(0,0,0,0.06) | 像一张纸轻放在桌上 |
| 按钮（主操作） | 24px（全圆角） | 0 2px 8px rgba(255,138,101,0.3) | 柔软可触，暗示"按我" |
| 输入框 | 20px | 内阴影 inset 0 2px 4px rgba(0,0,0,0.04) | 像纸面的凹陷 |
| 风格选择卡片 | 12px | 0 2px 6px rgba(0,0,0,0.08) | 可拿起的小卡片感 |
| 底部导航 | 0（与屏幕融合） | 0 -2px 8px rgba(0,0,0,0.04) | 不抢主体内容的注意力 |

### 2.3 字体规范

| 用途 | 字体 | 字号 | 字重 | 颜色 |
|------|------|------|------|------|
| 大标题 | 思源圆体 | 24px | Bold | #3E2723 |
| 小标题 | 思源圆体 | 18px | Medium | #3E2723 |
| 正文 | 苹方/思源黑体 | 15px | Regular | #795548 |
| 辅助说明 | 苹方/思源黑体 | 13px | Regular | #BCAAA4 |
| 按钮文字 | 思源圆体 | 16px | Medium | #FFFFFF |
| 宝宝语录（漫画中） | 手写体/站酷小薇 | 14-18px | - | #3E2723 |

**设计意图：** 使用圆体作为标题字体，传递"圆润、温暖、童趣"的品牌感。

### 2.4 图标风格

```
风格：圆润线性图标 + 微填充
线条粗细：2px
端点：圆角端点
颜色：跟随文字色或主题色

示例：
🏠 首页   → 圆角小房子，烟囱有心形烟
👶 角色库  → 圆脸宝宝轮廓
🎨 风格馆  → 圆形调色盘
📖 成长轴  → 打开的绘本
👤 我的    → 圆润人物剪影
```

## 三、信息架构（张小龙层）

### 3.1 页面层级结构

```
设计原则：最多2级深度，核心路径0分叉

宝贝漫记 (小程序)
│
├── Tab1: 创作 (首页/默认页)
│   ├── 文字输入
│   ├── 风格选择
│   ├── 生成等待
│   └── 作品展示 → 分享/保存
│
├── Tab2: 回忆 (成长时间轴)
│   ├── 时间线浏览
│   └── 单作品详情
│
├── Tab3: 角色库
│   ├── 角色列表
│   ├── 创建角色 (上传→生成→确认)
│   └── 角色卡详情
│
└── Tab4: 我的
    ├── 会员/额度信息
    ├── 家庭圈管理
    └── 设置
```

### 3.2 核心路径时间分配（30秒黄金路径）

```
时间线：
0s ─────── 5s ─────── 12s ─────── 15s ─────── 23s ─── 30s
│          │           │            │            │       │
打开       开始输入    输入完成      选好风格     出图     分享
小程序     文字/语音   点击下一步    点击生成     看到漫画  完成

各步骤设计要求：
├── 0-5s:  无启动页/无弹窗，直接进入创作页
├── 5-12s: 输入框自动聚焦，键盘弹起或语音按钮就绪
├── 12-15s: 风格卡片横滑，默认已选AI推荐项
├── 15-23s: 等待页有趣味动画，不觉得漫长
└── 23-30s: 漫画滑入展示，分享按钮最醒目
```

### 3.3 导航设计

```
┌─────────────────────────────────────┐
│          (内容区域)                   │

│                                     │
│                                     │
│                                     │
│                                     │
│                                     │
├─────────────────────────────────────┤
│                                     │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐  │
│  │ ✏️  │ │ 📖  │ │ 👶  │ │ 👤  │  │
│  │ 创作 │ │ 回忆 │ │ 角色 │ │ 我的 │  │

│                                     │
└─────────────────────────────────────┘

设计要点：
- 4个Tab足矣（张小龙：微信也只有4个Tab）
- "创作"Tab的图标比其他大20%，因为它是核心
- 进入创作流程后，底部Tab隐藏（沉浸式创作）
- Tab图标无文字标签（Jony Ive：图标本身足够表意时不需要文字）
  → 但首次使用时有引导气泡
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   → 点击可直接填入，降低起步门槛

④ 语音按钮在左下角
   → 带娃时左手抱娃右手操作
   → 长按说话，松手即转文字

⑤ 按钮文案"开始创作漫画"而非"生成"
   → "创作"让用户有参与感和创造感

情感化设计节点（Don Norman）：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
① 画笔动画而非普通loading
   → 暗示"有人在为你画画"（拟人化）
   → 等待变成了"观看创作过程"

② 进度条有明确百分比
   → Don Norman反馈原则：永远告知用户状态
   → 消除"要等多久"的焦虑

③ 随机创作小贴士
   → 等待时间不浪费，教用户下次写得更好
   → 变"等待"为"学习"

④ "小土豆的第7幅漫画即将诞生"
   → 个性化（用角色名字）
   → 计数器给成就感（已经画了6幅了！）
   → "诞生"这个词 → 像在期待新生命

⑤ 整个页面沉浸式，无导航无干扰
   → 创造"仪式感"
   → 像在等一份礼物被包装好
```

   → 进度条上方有小圆点标记每张漫画的位置
   → 拖动时显示对应漫画的日期和内容预览
   → 像在"翻阅时光"
```

### 5.8 视频功能设计验证标准

| 检查项 | 标准 | 对应设计层 |
|--------|------|-----------|
| 看到入口想点吗？ | "生成回忆视频"按钮点击率 >15% | 本能层 |
| 选模板纠结吗？ | 从进入到确认模板 <20秒 | 行为层 |
| 等待焦虑吗？ | 停留在等待页的用户 >60%不离开 | 行为层 |
| 看完视频感动吗？ | 完整播放率 >80% | 反思层 |
| 看完想分享吗？ | 播放后30秒内分享率 >40% | 反思层 |
| 发到视频号吗？ | 视频号发布率 >10% | 反思层 |
| 下次还想做吗？ | 30天内二次生成率 >30% | 反思层 |
